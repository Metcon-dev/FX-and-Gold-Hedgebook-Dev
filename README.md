# J2 Hedging Platform

J2 is a Flask + React platform for FX and metals hedging operations.

## Architecture
| Layer | Path | Purpose |
|---|---|---|
| Frontend | `j2-platform/client` | React UI (tabs, analytics views, trade workflows) |
| Backend API | `j2-platform/server/server.py` | Flask API routes, auth, sync orchestration, reporting |
| Domain models | `models/` | SQLite schema/bootstrap and trade data operations |
| Services | `services/` | PMX/StoneX integration, TradeMC sync, clean data pipeline |

## Data Storage
| File | Purpose |
|---|---|
| `fx_trading_ledger.db` | Main ledger DB (trades, TradeMC tables, auth users, scheduler state) |
| `pmx_database.db` | PMX operational DB (PMX trades, opening balances) |
| `j2_platform_clean.db` | Integrity/clean warehouse generated by clean pipeline |
| `manual_trades.json` | Backup/restore of manual trade number assignments |

## Data Sources
- PMX (StoneX) API endpoints for deal reports, account statements, balances, fixing invoices.
- TradeMC Directus API for trades, companies, and weight transactions.
- Manual user input from UI/API (trade tags, opening balances, export actions).

## Current Runtime Status
- Streamlit stack is removed/deprecated.
- Active stack is Flask API + React client.

## Local Development
### Backend (Flask)
1. Activate virtual environment.
2. Install dependencies from `j2-platform/server/requirements.txt` (and root requirements if needed).
3. Run API:
   - `python j2-platform/server/server.py`

### Frontend (React)
1. `npm --prefix j2-platform/client ci`
2. `npm --prefix j2-platform/client run dev`

## Quality Checks
- Frontend build:
  - `npm --prefix j2-platform/client run -s build`
- Backend smoke:
  - `python -m py_compile j2-platform/server/server.py`
  - `python -m py_compile services/trademc_service.py`
  - `python -m py_compile services/rest_service.py`
